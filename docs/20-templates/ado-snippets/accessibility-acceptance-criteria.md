## Accessibility acceptance criteria
- Keyboard: all interactive elements are reachable and operable using keyboard only.
- Focus: focus order is logical; focus indicator is visible.
- Screen reader: controls have accessible name/role/value; dynamic updates are announced.
- Errors: errors are announced and associated with the relevant field; recovery guidance is provided.
- Color/contrast: information is not conveyed by color alone; contrast meets requirements.
- States: empty/loading/error/success states are accessible.
